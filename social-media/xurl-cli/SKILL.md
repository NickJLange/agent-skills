# xurl — Official X API CLI

The official curl-like CLI for the X API. Supports OAuth 2.0 PKCE, OAuth 1.0a, and bearer token auth. Multi-app support with YAML config.

## Install

Download from GitHub releases (no sudo needed):
```
# Binary from latest release, install to ~/.local/bin
# See https://github.com/xdevplatform/xurl/releases
```

Also available via npm (`npm install -g @xdevplatform/xurl`) or Go (`go install github.com/xdevplatform/xurl@latest`).

## Setup

Config stored at `~/.xurl` (YAML). Register an app:

```bash
xurl auth apps add my-app --client-id CLIENT_ID --client-secret CLIENT_SECRET
xurl auth default my-app
```

## Auth Methods

### Bearer token (reads only)
```bash
xurl auth app --bearer-token YOUR_BEARER
xurl --auth app /2/users/me
```

### OAuth 2.0 PKCE (full access)
Requires redirect URI `http://localhost:8080/callback` set in developer portal.

```bash
xurl auth oauth2
```

**Headless/server (important gotchas):**
- Always set the correct app first: `xurl auth default <app-name>`
- Do **not** run multiple concurrent `xurl auth oauth2` sessions
- The callback `state`/`code` must match the exact URL generated by that same in-flight session
- If `client_id=` is blank in the URL, default app is wrong (often set to `default` with empty creds)

Recommended flow (background listener):

```bash
xurl auth default my-app
xurl auth oauth2 > /tmp/xurl_oauth.txt 2>&1 &
sleep 2
cat /tmp/xurl_oauth.txt
# Send the printed URL to user immediately; user authorizes in browser
# User pastes full callback URL back (includes code + state)
```

**Fallback: Manual PKCE exchange (when background listener fails)**

If `xurl auth oauth2` doesn't catch the callback (common on headless), use a manual Python approach. The problem: xurl generates the PKCE verifier internally and never exposes it, so you can't exchange the code after the fact.

Manual flow:

```python
import hashlib, secrets, base64, urllib.parse, urllib.request, json

CLIENT_ID = "YOUR_CLIENT_ID"
CLIENT_SECRET = "YOUR_CLIENT_SECRET"
REDIRECT_URI = "http://localhost:8080/callback"

# 1. Generate PKCE pair
code_verifier = secrets.token_urlsafe(64)[:64]
code_challenge = base64.urlsafe_b64encode(
    hashlib.sha256(code_verifier.encode()).digest()
).rstrip(b"=").decode()

state = secrets.token_urlsafe(32)
scopes = "tweet.read users.read bookmark.read follows.read list.read block.read mute.read like.read users.email dm.read tweet.write tweet.moderate.write follows.write bookmark.write block.write mute.write like.write list.write media.write dm.write offline.access space.read"

# 2. Build auth URL — send to user
params = urllib.parse.urlencode({
    "client_id": CLIENT_ID,
    "code_challenge": code_challenge,
    "code_challenge_method": "S256",
    "redirect_uri": REDIRECT_URI,
    "response_type": "code",
    "scope": scopes,
    "state": state,
})
auth_url = f"https://x.com/i/oauth2/authorize?{params}"
# SAVE code_verifier and state — you need them for step 3

# 3. After user pastes callback URL, extract code and exchange
# (extract code from the callback URL's ?code= parameter)
import base64
token_data = urllib.parse.urlencode({
    "code": AUTH_CODE,
    "grant_type": "authorization_code",
    "client_id": CLIENT_ID,
    "redirect_uri": REDIRECT_URI,
    "code_verifier": code_verifier,
}).encode()

creds = base64.b64encode(f"{CLIENT_ID}:{CLIENT_SECRET}".encode()).decode()
req = urllib.request.Request(
    "https://api.x.com/2/oauth2/token",
    data=token_data,
    headers={
        "Content-Type": "application/x-www-form-urlencoded",
        "Authorization": f"Basic {creds}",
    },
)
resp = urllib.request.urlopen(req)
tokens = json.loads(resp.read())  # access_token + refresh_token
```

**Critical:** X API requires Basic auth header (`client_id:client_secret`) for token exchange. Sending just `code_verifier` as a "public client" returns 401. Both client_secret AND code_verifier must be present.

**xurl config parser gotcha (confirmed):**
- The YAML key for OAuth2 tokens is `oauth2_tokens` (found via `strings ~/.local/bin/xurl` — Go struct tag `yaml:"oauth2_tokens,omitempty"`)
- Even with correct YAML structure (access_token + refresh_token under oauth2_tokens), xurl still shows `oauth2: (none)` in `auth status`
- Tried multiple formats: `oauth2`, `oauth2_tokens`, with/without `type`, `token_type`, `expiration_time` — all fail silently
- **Conclusion:** xurl's YAML parser expects a Go-serialized struct that can't be hand-edited. The `SaveOAuth2Token` function writes to a different internal representation.
- **Use `/opt/data/scripts/xapi.py` wrapper instead** — direct Python urllib calls with the bearer token. Fully functional for reads, writes, bookmarks.

**Token storage (upgrade-proof):**
- Tokens saved at `/opt/data/config/x-oauth2-tokens.json` (our config dir, survives tool reinstalls)
- xurl's `~/.xurl` config parser is finicky about oauth2 token format — manual injection often shows `(none)` even with correct YAML
- **Workaround:** Use direct Python API calls with the access token. xurl binary can be upgraded without losing tokens since they live in `/opt/data/config/`
- Token refresh endpoint: `POST https://api.x.com/2/oauth2/token` with `grant_type=refresh_token`, `refresh_token=TOKEN`, `client_id`, `client_secret` (Basic auth)

**Client secret mismatch gotcha:** If you get 401 "unauthorized_client" with your app's client_secret, check that the secret in `~/.xurl` matches what's in the X Developer Portal. The portal may show a different secret than what xurl has stored — regenerate and update if needed.

If repeated "You weren’t able to give access to the App" appears, verify in portal:
- app is attached to a Project with API access
- callback URI exactly matches `http://localhost:8080/callback`
- app auth settings (type/permissions/required URLs like website/privacy/ToS) are complete
- client secret is current (regenerate and update app config if needed)

### OAuth 1.0a
```bash
xurl auth oauth1 --consumer-key KEY --consumer-secret SECRET --access-token TOKEN --token-secret SECRET
```

## Key Findings

- X API v2 lists endpoints work with bearer token only (no OAuth needed)
  - GET /2/users/:id/owned_lists
  - GET /2/lists/:id/tweets
- For bookmarks/posting/likes, OAuth 2.0 or OAuth 1.0a user auth is required
- OAuth 2.0 client ID/secret are NOT the same as OAuth 1.0a consumer key/secret or access tokens
- On the X Developer Portal, "OAuth 1" = OAuth 1.0a
- xurl is preferred over x-cli (third-party) because it supports OAuth 2.0 PKCE flow

## Common Requests

```bash
# User lookup
xurl --auth app /2/users/by/username/openai

# Search tweets
xurl --auth app "/2/tweets/search/recent?query=AI+agents&max_results=5"

# Bookmarks (needs OAuth2)
xurl --auth oauth2 /2/users/me/bookmarks

# Post tweet (needs OAuth1 or OAuth2)
xurl --auth oauth2 -X POST /2/tweets -d '{"text":"hello"}'
```

## Multi-App

```bash
xurl auth apps list
xurl auth status
xurl --app other-app /2/users/me
```
